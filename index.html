Description of Group Meeting Activity

The project aims to assess renewable energy availability at the GLEAMM facility and implement workload control using an API on the test cluster's head node. The focus is on retrieving real-time power data, predicting available power, and dynamically scheduling jobs on the test cluster to align with renewable sources and backup battery levels. The API can also remotely power on/off nodes, optimizing energy usage and extending battery run time as needed.

{Stage 2: The project aims to collect data on renewable energy availability at the GLEAMM facility and develop an API to control workloads on a test cluster's head node. This involves monitoring available power, planning power usage based on web data, and scheduling jobs on the cluster to match renewable power and backup battery capacity. The API can also power on/off nodes to minimize energy usage when necessary and extend battery life.}

{Stage 3: We were unable to access the GLEAMM cluster through the API. Nevertheless, the application is designed with the capability to connect to a functional API, enabling remote stress_ng/load management functionality on the cluster.}

Benefits

This project can provide the benefits of optimizing operational costs and minimizing carbon emissions by implementing an innovative solution.
Also, the system is designed to be scalable and capable of meeting the demands of large-scale data facilities.
Through intelligent workload control, the project ensures the efficient utilization of renewable energy sources, contributing to environmental sustainability.
Moreover, the implementation allows for the testing of cluster operation limits, enabling a comprehensive evaluation of system capabilities.
